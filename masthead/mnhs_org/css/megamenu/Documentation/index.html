Upgrading the Menu

As the version 4.0 brings major changes, here's a quick guide to upgrade from the version 3.32.

First you'll need to replace these script files :

  • megamenu_plugins.js
  • megamenu.js (or megamenu.min.js if you use the minified version)

In megamenu.css, only the section '8. Form Elements' has been changed due to the addition of the contact form, nothing else was modified.

If you plan to use the contact form, you'll also need to add the file contact.php and add your email address.

Quick Menu Setup

To start integrating this menu, there are 4 necessary files :

  • css/megamenu.css
  • js/jquery.js
  • js/megamenu_plugins.js
  • js/megamenu.js

From the version 3.0, IE6 is not supported anymore.

In your HTML file, just make sure that you link properly to each of the above files.

In the folder img, there are a few images that should be included too, the 6 photo placeholders are only illustrative and all other graphics are used to style the menu or its content.

Now it's time to add the HTML : in the package, there are 5 HTML files all showing a possible use / variant of the menu. You can choose either the horizontal or the vertical menu, they use exactly the same CSS and javascript but the HTML markup is slightly different : there are 2 specific classes applied to the menu containers and an additional holder for each drop down.

	<div class="megamenu_container megamenu_dark_bar megamenu_light">

	    <ul class="megamenu">

	    MEGA MENU CONTENT

	    </ul>

	</div>

And from the version 3.2, you can use a fixed version of the menu. It'll stick to the top of the browser when you'll use this code :

	<div class="megamenu_container megamenu_fixed megamenu_dark_bar megamenu_light">

	    <ul class="megamenu">

	    MEGA MENU CONTENT

	    </ul>

	</div>

The above code is for the horizontal menu. To enable the vertical variant, you'll use this code :

	<div class="megamenu_container_vertical megamenu_dark_bar megamenu_light">

	    <ul class="megamenu megamenu_vertical">

	    MEGA MENU CONTENT

	    </ul>

	</div>

Menu Color Variants

For both variants (horizontal and vertical), you can simply change a few colors by modifying the classes of the menu container :

	<div class="megamenu_container megamenu_dark_bar megamenu_light">

You can first choose a color for the menu bar :

  • megamenu_dark_bar
  • megamenu_light_bar

Then the color of the drop downs :

  • megamenu_dark
  • megamenu_light

Those 2 options above are necessary, the elements won't be displayed properly if you don't use them.

The colors that ware available in the previous versions of the menu are still available, and to use them, all you have to do is to add another class to the menu container. For example if you want to set the blue color, the code will be :

	<div class="megamenu_container megamenu_dark_bar megamenu_light megamenu_bluetheme">

Applying one of these classes will only modify the menu bar color and the color of the links / list elements. You can choose between the following classes :

Menu Effects, Javascript Files

Before you include the javascript files into your HTML pages, make sure that you don't load jQuery several times, it's unnecessary and can cause issues.

This Mega Menu comes with 9 jQuery-powered effects which are applied as following (it's the script that you'll find at the top of all HTML files) :

	<script type="text/javascript" src="js/jquery.js"></script>
	<script type="text/javascript" src="js/megamenu_plugins.js"></script>
	<script type="text/javascript" src="js/megamenu.js"></script>
	<script type="text/javascript">
	$(document).ready(function($){
	    $('.megamenu').megaMenuCompleteSet({
	        menu_speed_show : 300, // Time (in milliseconds) to show a drop down
	        menu_speed_hide : 200, // Time (in milliseconds) to hide a drop down
	        menu_speed_delay : 200, // Time (in milliseconds) before showing a drop down
	        menu_effect : 'hover_fade', // Drop down effect, choose between 'hover_fade', 'hover_slide', etc.
	        menu_click_outside : 1, // Clicks outside the drop down close it (1 = true, 0 = false)
	        menu_show_onload : 0 // Drop down to show on page load (type the number of the drop down, 0 for none)
	    });
	});
	</script>

The first line loads jQuery, as said above, if you already use it in your pages, avoid loading it several times.

The second line links to the mega menu script plugins (containing HoverIntent and Modernizr) followed by the megamenu script (both should be placed in this order after the jquery framework), and the small code initializes the mega menu by applying an "hover fade" effect (fading the drop down on mouse hover) with a few other options. To change the effect, simply replace "hover_fade" by one of the following values :

  • hover_fade
  • hover_slide
  • hover_toggle
  • click_fade
  • click_slide
  • click_toggle
  • open_close_fade
  • open_close_slide
  • open_close_toggle

Here are some explanations of those effects :

  • hover_fade : fade effect on mouse hover
  • hover_slide : slide effect on mouse hover
  • hover_toggle : toggle effect on mouse hover
  • click_fade : the drop down fades in on mouse click (and remains visible as long as the mouse hovers on it)
  • click_slide : the drop down slides on mouse click (and remains visible as long as the mouse hovers on it)
  • click_toggle : toggle effect on mouse click (and the drop down remains visible as long as the mouse hovers on it)
  • open_close_fade : 1 click to open, 1 click to close (fade effect)
  • open_close_slide : 1 click to open, 1 click to close (slide effect)
  • open_close_toggle : 1 click to open, 1 click to close (toggle effect)

In the script that appears within the HTML pages, you also have a few other options (explained by the comments) :

  • menu_speed_show
  • menu_speed_hide
  • menu_speed_delay
  • menu_effect
  • menu_click_outside
  • menu_show_onload

Only the 2 last options are not made to be combined with all the effects, they work only with : open_close_fade, open_close_slide, open_close_toggle. That's because those effects show the drop downs and keep them visible until the user clicks somewhere else (and all other effects require a mouse hover so once you hover outside the drop down, it's hidden.

By activating the option menu_click_outside, it closes any visible drop down by clicking outside it (or anywhere else on the page).

The following option, menu_show_onload, can be used by entering the number of the drop down you want to see already visible when the page loads. If for example you want the visitors to see the 3rd drop down already opened when they reach your site, just put the value '3' there.

There's also a minified version of the mega menu script situated in the "js" folder.

HTML Structure

In the package, you'll find 5 coded HTML pages. Those files will show you different possibilities in terms of typography and some columns variants, there are many more possibilities and this menu is quite flexible so feel free to create your own layouts.

Since the version 3.0, all the menu works with percentages instead of pixel values. It means that the menu (and all drop downs) will expand to the size of their container. In all the page examples, you'll see the menu wrapped into the page_wrapper container. This element doesn't have to be used in your pages, all you have to do is to drop the whole menu into your own container and all elements will fit according to the space they'll have.

Just for testing, you can modify the width of the page_wrapper element at the beginning of the CSS to see how the menu fits into different contexts.

Setting the drop downs containers

Once you have the menu containers in place (see the section 'Quick Menu Setup'), you can now add menu elements with or without drop downs. Let's start with our previous markup, as you can see, the menu is made of an unordered list. The list elements will appear in the menu bar and they can contain drop downs :

	<div class="megamenu_container megamenu_dark_bar megamenu_light">

	    <ul class="megamenu">

		    <li><a href="#" class="megamenu_drop">Home</a>

		    FIRST DROP DOWN

		    </li>

		    <li><a href="#" class="megamenu_drop">About</a>

		    SECOND DROP DOWN

		    </li>

		    <li><a href="#" >No Drop Down</a></li>

	    </ul>

	</div>

Within these list elements, you have the choice for your drop downs, from the smallest to the full width option. For our example, we'll keep only the first element markup from te above code :

	<li><a href="#" class="megamenu_drop">Home</a>

	FIRST DROP DOWN

	</li>

Here we have the possibility to use any of the 13 drop downs like so :

	<li><a href="#" class="megamenu_drop">Home</a>

		<div class="dropdown_fullwidth">

		</div>

	</li>

dropdown_fullwidth is the largest option available and will expand to the whole menu bar width. As a side note, dropdown_12columns works in the same way, it's something I kept from the earlier versions of the menu. You can use one or the other in the same way.

To set a drop down that will have half of the menu bar width, we use :

	<li><a href="#" class="megamenu_drop">Home</a>

		<div class="dropdown_6columns dropdown_container">

		</div>

	</li>

IMPORTANT NOTICE : if you use any other container than dropdown_fullwidth, you have to use the class dropdown_container as shown in the above example. This class is used by the script and the CSS and the menu won't work properly if you don't use this class.

Here's the full list of the containers available (that require the dropdown_container class) :

  • dropdown_1column
  • dropdown_2columns
  • dropdown_3columns
  • dropdown_4columns
  • dropdown_5columns
  • dropdown_6columns
  • dropdown_7columns
  • dropdown_8columns
  • dropdown_9columns
  • dropdown_10columns
  • dropdown_11columns
  • dropdown_12columns
  • dropdown_fullwidth

The class names used don't refer to the number of columns that they can hold, it's just a reference to order them by size. In other words, you can put 12 columns in any of them. Of course adding 12 columns in the smallest container doesn't make sense but the idea behind this structure is to keep a maximum flexibility.

To understand better how to use these classes, imagine that :

  • dropdown_12columns is the largest and dropdown_1column is 1/12 of dropdown_12columns,
  • dropdown_2columns is 2/12 of dropdown_12columns,
  • dropdown_3columns is 3/12 of dropdown_12columns,.
  • ...
  • dropdown_6columns is 6/12 (or one half) of dropdown_12columns, etc.

Setting columns within the drop downs

Within the above containers, you can use from 1 up to 12 columns but don't forget to check if your container is large enough for the columns you include. It will work in any container but keep in mind that the drop down containers are proportionnal to the menu bar width. Adding 12 columns within a small container doesn't make sense.

There are 12 column choices, from the smallest to the largest. In the same way, you have to think these columns in terms of proprtion to their container. Let's use our previous markup and add a few columns :

	<li><a href="#" class="megamenu_drop">Home</a>

		<div class="dropdown_6columns dropdown_container">

			<div class="col_12">

			</div>

			<div class="col_8">

			</div>

			<div class="col_4">

			</div>

		</div>

	</li>

The classes that can be used are :

  • col_12
  • col_11
  • col_10
  • col_9
  • col_8
  • col_7
  • col_6
  • col_5
  • col_4
  • col_3
  • col_2
  • col_1

col_12 is the largest and col_1 the smallest. Now you have to keep in mind that to create a row of content, you need to reach a value of 12 by adding the numbers used in these class names. For example, col_12 will expand to the whole drop down space available. If you add other columns, they'll fall under this one.

In the sample above, I've used col_12 and the second row of content has col_8 combined with col_4 : 8 + 4 = 12, together they exand to the whole space available. These column containers are using percentages so you can think of it with this system :

  • col_12 is 100%
  • col_11 is 11/12 of 100%
  • col_10 is 10/12 of 100%
  • col_9 is 9/12 of 100%
  • col_8 is 8/12 of 100%
  • col_7 is 7/12 of 100%
  • col_6 is 6/12 of 100%
  • col_5 is 5/12 of 100%
  • col_4 is 4/12 of 100%
  • col_3 is 3/12 of 100%
  • col_2 is 2/12 of 100%
  • col_1 is 1/12 of 100%

Fly-Out Menu

Since the version 3.0, this menu includes a multi-level fly-out menu. If you open 01_megamenu_example1.html and hover over 'Fly-Out', you'll see a 4-levels fly-out drop down. You can add as many levels as you want but the script won't keep them inside the viewport : if you don't want some levels to fall outside the visible area, you have to manually set their alignment, they can be displayed towards the left or the right.

This fly-out menu is made of nested unordered lists and use the same jQuery effect as the mega drop downs. It is not possible to set a different effect for each type of drop down.

Here's the basic structure of a fly-out menu :

	<li><a href="#" class="megamenu_drop">Fly-Out</a>

		<div class="dropdown_2columns dropdown_container">

			<ul class="dropdown_flyout">
                    
				<li><a href="#">Level 1</a></li>
                        
				<li class="dropdown_parent"><a href="#">Level 1</a>

					<ul class="dropdown_flyout_level">

						<li><a href="#">Level 2</a></li>
						<li><a href="#">Level 2</a></li>
						<li><a href="#">Level 2</a></li>

					</ul>

				</li>

				<li><a href="#">Level 1</a></li>
				<li><a href="#">Level 1</a></li>
				<li><a href="#">Level 1</a></li>

		</div>

	</li>

Now to simply display the level 2 towards the left, all you need to do is to add the class dropdown_flyout_level_left :

	<li><a href="#" class="megamenu_drop">Fly-Out</a>

		<div class="dropdown_2columns dropdown_container">

			<ul class="dropdown_flyout">
                    
				<li><a href="#">Level 1</a></li>
                        
				<li class="dropdown_parent"><a href="#">Level 1</a>

					<ul class="dropdown_flyout_level dropdown_flyout_level_left">

						<li><a href="#">Level 2</a></li>
						<li><a href="#">Level 2</a></li>
						<li><a href="#">Level 2</a></li>

				</li>

				<li><a href="#">Level 1</a></li>

				<li><a href="#">Level 1</a></li>

		</div>

	</li>

Vertical Menu

The vertical version of this menu works like the horizontal version with some minor differences, and you can switch between the 2 versions just by doing some modifications in the markup.

Firstly, the vertical menu uses 2 additional classes as explained in the 'Quick Menu Setup' section :

	<div class="megamenu_container_vertical megamenu_dark_bar megamenu_light">

	    <ul class="megamenu megamenu_vertical">

	    MEGA MENU CONTENT

	    </ul>

	</div>

As you can see, there are 2 specific classes used for the vertical menu : megamenu_container_vertical and megamenu_vertical.

The second difference is that the drop downs require an additional container, as explained in the section 'HTML structure', the basic markup for a menu element and its drop down is :

	<li><a href="#" class="megamenu_drop">Home</a>

		<div class="dropdown_6columns dropdown_container">

		DROP DOWN CONTENT

		</div>

	</li>

The vertical menu requires an additional container as following :

	<li><a href="#" class="megamenu_drop">Home</a>

		<div class="dropvertical_container">

			<div class="dropdown_6columns dropdown_container">

			DROP DOWN CONTENT

			</div>

		</div>

	</li>

That's all you need to be able to use the vertical mega menu, the CSS and javascript are the exact same files.

Mobile Devices

This menu has been coded and tested to work on all kinds of mobile devices such as iPhone or iPad. On those devices, all jQuery effects are disabled and the menu works in the same way whatever the effect you choose : the drop downs are opened and closed with the "touch" event.

Due to the large variety of mobile devices available, it is not possible for me to test this menu on all of them so I've tried to make things as simple as possible. If by any chance you see an issue, feel free to contact me and I'll do my best to fix the problem.

You don't have to do anything to make this menu mobile friendly, the javascript and the CSS will do the job for you. There is however an additional option that can be useful : if you open 01_megamenu_example1.html and resize your browser until you reach a width which is less than 767px, you'll see that all menu items are stacked one under the other. All drop down columns fill the whole space available instead of being aligned one next to the other. Now if you hover over 'Home', you'll see that at the bottom of the drop down, the lists and the icons are still split into columns : when you create your columns as explained in the section 'HTML Structure', you have additional classes that keep your content organized into columns even in the mobile version. This is convenient when your content is not too large and can fit into columns instead of taking the whole space available.

You have 4 options available : you can have your content split into :

  • 6 columns with the class responsive_sixths
  • 4 columns with the class responsive_fourths
  • 3 columns with the class responsive_thirds
  • 2 columns with the class responsive_halfs

Those classes must be added directly to each column container, let's say you have a drop down with 3 equal columns, the markup will look like :

	<li><a href="#" class="megamenu_drop">Responsive</a>

		<div class="dropdown_6columns dropdown_container">

			<div class="col_4">

			</div>

			<div class="col_4">

			</div>

			<div class="col_4">

			</div>

		</div>

	</li>

Now let's say that you want to keep 3 columns on mobile devices, you'll need to use the class responsive_thirds as following :

	<li><a href="#" class="megamenu_drop">Responsive</a>

		<div class="dropdown_6columns dropdown_container">

			<div class="col_4 responsive_thirds">

			</div>

			<div class="col_4 responsive_thirds">

			</div>

			<div class="col_4 responsive_thirds">

			</div>

		</div>

	</li>

Contact Form

Since the version 4.0, this menu includes a working contact form ready to be used. All you have to do is to add your email address in the file contact.php at the line 3. This contact form has been tested and sends emails without any problem, be sure to check your spam folder if you don't see anything coming to your inbox.

If you need to change or translate the feedback messages (when a field is blank or improperly filled), open the file megamenu.js and find this code :

    messages: {
        name: {
            required: "Please enter your name",
            minlength: "Your name should contain at least 2 characters"
        },
        email: {
            required: "Please enter your email address"
        },
        message: {
            required: "Please enter your message",
            minlength: "This is too short"
        },
        captcha: {
            required: "Wrong answer !"
        }
    },

If you make any change, be sure to leave the double quotes, if you remove them, the form may not work as expected.

You can also define options for each field like making a field optionnal or required or define a minimum length. In this same file megamenu.js, you'll find the options there :

    rules: {
        name: {
            required: true,
            minlength: 2
        },
        email: {
            required: true,
            email: true
        },
        message: {
            required: true
        },
        captcha: {
            required: true,
            answercheck: true
        }
    },

You can find more options in the jQuery validation plugin page.

CSS Files and Structure

In the css folder, you'll find the only file required : megamenu.css. It contains all the required (and optional) styling for the mega menu.

If you want to modify some properties in the CSS, make sure that you keep a safe copy of the original file before starting. This CSS is quite complex and requires a good knowledge of CSS markup to be safely modified.

This file is divided into 9 main parts and modifying this file may affect the functionnality so be very careful when you start your customizations :

  • 01 MEGA MENU BAR : It's the code for the menu container and the menu items (the entire menu bar).
  • 02 MEGA MENU CONTAINERS : This part defines all the drop down containers that will hold the columns.
  • 03 DROP DOWN COLUMNS GRID : Here you'll find the column grid system, everything is in percentages.
  • 04 FLY-OUT MENU : Instead of mega drop downs, you can use regular fly-out menus with infinite sub-levels.
  • 05 MEGA MENU WEB ELEMENTS : This part contains styling for elements like headers, lists, forms, images, social icons, etc.
  • 06 DEFAULT THEMES, DARK & LIGHT (REQUIRED) : In this part, all colors styling the menu bar and the drop downs are defined. You have 2 default themes, dark or light that can be combined : you can have a dark menu bar and light drop downs, or the opposite.
  • 07 ADDITIONAL THEMES (OPTIONNAL) : These color schemes are totally optionnal, there are 13 options available that style the menu bar color and the links / list elements.
  • 08 VERTICAL MEGA MENU : This part contains specific markup used in the vertical mega menu variant.
  • 09 MOBILE DEVICES : Here are all the necessary media queries making this menu completely responsive.
